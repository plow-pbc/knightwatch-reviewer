# Plow

Plow is an agent platform that gives OpenClaw agents narrow, permission-gated access to cloud services (Gmail, Slack) and on-device capabilities through a local approval layer.

The system has three tiers:

- **Plow API** (`api/`) — cloud service that owns connector OAuth, opportunity matching, marketplace skills, and policy enforcement
- **Plow app** (`app/`) — macOS app (Swift) + plowd daemon (Python) that manages on-device approval, gateway configuration, and VM lifecycle
- **Plow CLI** (`cli/`) — narrow tool surface inside the OpenClaw container, talks directly to the Plow API

## Architecture

**Data flow:**

- Inbound SMS → Linq → Plow API webhook → WebSocket → app → LLM → outbound SMS ([full routing doc](docs/architecture/sms-inbound-routing.md))
- `plow gmail search "mortgage"` → CLI → Plow API → Gmail
- `plow slack search "renewal"` → CLI → Plow API → Slack
- OpenClaw never holds raw OAuth tokens — it sees narrow CLI tools only

# Plow App

The Plow macOS app is currently a local OpenClaw wrapper/orchestrator centered on **PlowD** (the daemon).

## Current Focus

As of this branch, the primary path is:

- PlowD starts and manages OpenClaw gateway + node host locally.
- Plow app tooling keeps app-managed config/state under `~/Library/Application Support/co.plow.app/...` instead of `~/.openclaw/...`.
- Plow gateway policy/operator logic handles pairing and tool approval flows.

The older Wren/Inbox agent-first flow is not the main focus right now, and agent registration on daemon startup is currently disabled.

## Architecture (Now)

- **PlowD (FastAPI daemon)**: process manager + policy service + status API.
- **OpenClaw gateway**: local websocket gateway (`127.0.0.1`, token auth).
- **OpenClaw node host**: execution host process paired to the gateway.
- **OpenClaw dashboard/web chat**: optional local UI client that connects to the same gateway.
- **Plow macOS app**: still exists as iMessage bridge/UI layer, but not the center of this PR.

OpenClaw terms vs app terms:

- PlowD == daemon/operator-side process (not the gateway binary itself)
- OpenClaw gateway == websocket service PlowD launches/manages
- Web client/dashboard == separate operator client that can connect locally when paired/authenticated

State roots are bundle-scoped:

- production app: `~/Library/Application Support/co.plow.app/`
- default dev app: `~/Library/Application Support/co.plow.app.dev/`
- worktree/dev variants: `~/Library/Application Support/<bundle-id>/`

- state: `~/Library/Application Support/<bundle-id>/openclaw/gateway`
- token: `~/Library/Application Support/<bundle-id>/openclaw/secrets/gateway-token`

   527	/// Pure mapping from screen state + measured intrinsic heights → desired
   528	/// installer window height. Splash screens (welcomeInstall / welcomeFDA /
   529	/// activation / connectContext) keep the design height of 530 because they
   530	/// fill via `Spacer`s rather than intrinsic content. The connectors screen
   531	/// sizes to its inner SettingsView content (reported via
   532	/// `InstallerContentHeightKey`) plus the bottom-anchored DownloadBarView's
   533	/// height (reported via `InstallerOverlayHeightKey`, zero when absent), and
   534	/// is capped so a long settings list falls back to scrolling inside
   535	/// SettingsView's existing ScrollView rather than pushing the window
   536	/// off-screen.
   537	///
   538	/// Returns 530 (the splash height) for connectors when no measurement has
   539	/// arrived yet — better to render at the design height for one frame than
   540	/// flash a degenerate window.
   541	///
   542	/// The cap applies to *all* screens, including splash: on a degenerate
   543	/// `visibleFrame` (multi-display reconfiguration, sidecar with a tiny
   544	/// secondary display, etc.) the cap may drop below 530, in which case
   545	/// fitting the visible frame matters more than honoring the design height.
   546	func installerWindowHeight(
   547	    for screen: InstallerScreen,
   548	    contentHeight: CGFloat,
   549	    overlayHeight: CGFloat,
   550	    cap: CGFloat
   551	) -> CGFloat {
   552	    let splashHeight: CGFloat = 530
   553	    let unclamped: CGFloat = {
   554	        guard screen == .connectors, contentHeight > 0 else { return splashHeight }
   555	        return contentHeight + overlayHeight
   556	    }()
   557	    return min(unclamped, cap)
   558	}
